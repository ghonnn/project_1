1. Ringkasan Eksekutif

Menu Voucher adalah alat self-service bagi admin ISP untuk menjual akses internet hotspot berbasis kuota/waktu (model voucher MikroTik + FreeRadius). Seluruh alur—dari mendefinisikan paket harga sampai mencetak kode login—dilakukan dari satu grup menu, tanpa perlu masuk ke Winbox/terminal MikroTik untuk membuat user satu per satu.

Inti nilai produk:

  • Sekali setup, ribuan voucher. Definisikan profil paket satu kali, lalu generate voucher massal (s.d. 500 per batch).
  • Otomatis tersinkron ke FreeRadius. Profil → tabel radgroupreply; voucher → radcheck + radusergroup. Tidak ada input manual di sisi RADIUS.
  • Siap jual. Voucher bisa langsung dicetak (HTML grid) atau diekspor (CSV) untuk reseller/outlet.
  • Terlacak. Status stok → terjual → expired, plus pemantauan sesi online dan rekap komisi.
🎯 Tujuan dokumen
Menjadi PRD terbaru sekaligus SOP penggunaan menu Voucher: (1) cara membuat paket harga, (2) cara menghasilkan kode login hotspot, (3) apa yang terjadi di balik layar, dan (4) rekomendasi penyederhanaan untuk iterasi berikutnya.

4. Prasyarat Sebelum Mulai

  1. Tenant aktif. Pastikan tenant Anda terpilih (mis. NEX Demo ISP).
  2. Router terdaftar di menu Jaringan → Router. Diperlukan saat generate voucher.
  3. Radius Server terdaftar (FreeRadius RND: 10.20.1.19). Diperlukan agar voucher tahu server tujuan.
  4. Tabel FreeRadius SQL sudah dimigrasi. Jika belum, voucher tetap dibuat namun belum bisa login sampai sync ulang.
  5. MikroTik hotspot sudah mengarah ke FreeRadius (RADIUS client/NAS) dan memakai login.html dari menu Template.
⚠️ Penting
Nilai MikroTik Group pada Profile harus identik dengan group yang dikenali konfigurasi MikroTik Anda. Ketidakcocokan group = voucher valid di RADIUS tetapi tidak mendapat rate-limit/address-list yang benar.

ℹ️ Yang terjadi saat generate
generateVouchers() → HotspotVoucherService::generate() membuat N baris HotspotVoucher (status stock) dalam satu transaksi, lalu tiap voucher disinkron: radcheck (Cleartext-Password + Simultaneous-Use) dan radusergroup (user → group paket). Kolom Sync berubah menjadi Synced bila berhasil.
⏱️ Kapan voucher kedaluwarsa?
Masa Aktif Hari mulai dihitung saat voucher ditandai terjual (tombol Terjual atau penjualan), bukan saat dibuat. Jadi stok bisa disimpan lama tanpa hangus.

7. Langkah 3 — Cetak & Distribusi Voucher

Di menu Stok voucher, setelah generate tersedia dua aksi:

  • Print HTML — mengunduh nex-voucher-print.html: grid kartu voucher 3 kolom (username, password, batch, outlet, harga) yang otomatis membuka dialog cetak. Cocok untuk dipotong & dijual fisik.
  • Export CSV — mengunduh nex-hotspot-vouchers-*.csv (username, password, profile, router, server, partner, outlet, harga, status, batch) untuk reseller atau impor ke sistem lain.
💡 Best practice distribusi
Generate per outlet/partner terpisah (isi field Outlet) agar rekap komisi & rekonsiliasi stok per lokasi rapi. Gunakan satu Batch Code per gelombang cetak.

11. Best Practices Operasional

  • Konvensi penamaan profil & group. Selaraskan, mis. paket "1 Jam" → group NEX-HOTSPOT-1J. Konsisten memudahkan audit di MikroTik.
  • Prefix bermakna. Pakai prefix per lokasi/promo (mis. PASAR, EVENT) agar asal voucher terbaca dari username.
  • Password 6 karakter adalah sweet spot: cukup aman, mudah diketik. Hindari >8 untuk voucher cetak.
  • Generate per outlet. Isi Partner/Outlet saat generate demi rekap komisi otomatis.
  • Kuota vs Durasi. Paket berbasis waktu → set Durasi, Kuota = 0. Paket berbasis volume → set Kuota, Durasi besar/0.
  • Uji 1 voucher dulu sebelum cetak massal: generate 1, login di MikroTik, pastikan rate-limit & timeout benar.
  • Simpan stok, jual baru tandai. Karena masa aktif dihitung saat terjual, aman menumpuk stok.

12. Rekomendasi Penyederhanaan (Roadmap)

Sebagai PM, berikut usulan agar alur makin simpel & minim kesalahan. Prioritas dari dampak tertinggi.

1Auto-isi MikroTik Group dari Nama Profile (slug) dengan opsi override.Menghilangkan sumber error #1 (group mismatch).Tinggi
2Wajibkan Router & Radius Server + validasi saat generate (kini nullable).Mencegah voucher "yatim" tanpa NAS tujuan.Tinggi
3Pindahkan Harga/HPP/Komisi ke kolom tabel tersendiri (kini disimpan dalam JSON attributes profil).Mempermudah query rekap & laporan margin yang akurat.Sedang
4Helper text in-form: "Kuota 0 = unlimited", contoh format Rate Limit.Mengurangi salah isi tanpa pelatihan.Sedang
5Tombol "Generate & Print" satu klik.Memangkas langkah untuk operator outlet.Sedang
6Filter & pencarian pada tabel stok/terjual (status, batch, outlet).Skala ribuan voucher butuh pencarian.Sedang
7Aksi massal "Sinkron ulang" untuk voucher Pending.Pemulihan cepat saat RADIUS sempat down.Rendah

Catatan: alur saat ini sudah fungsional & koheren; usulan di atas bersifat penyempurnaan, bukan perbaikan bug.

13. Troubleshooting & FAQ

GejalaPenyebab & Solusi
Voucher dibuat tapi kolom Sync = PendingTabel FreeRadius belum dimigrasi. Jalankan migrasi RADIUS, lalu generate ulang/sinkron.
Login diterima tapi kecepatan/limit salahMikroTik Group di profil tidak cocok dengan konfigurasi MikroTik. Samakan nama group.

Halaman login polos/tanpa brandinglogin.html dari menu Template belum diunggah ke MikroTik.
Voucher cepat hangusCek Masa Aktif Hari; ingat penghitungan mulai saat ditandai terjual.
